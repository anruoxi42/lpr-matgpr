# KJ 地质建模算法深度分析报告

## 分析对象

`src/processing/algorithms.js` — `geologicModel()` 函数（行 1620-1789）

## 算法逐步骤分析

### 步骤 1：预处理流水线（行 1631-1635）

```js
r = removeDC  →  freqFilter(bp)  →  backgroundRemove  →  slidingBackground  →  agc(gaussian)
```

**问题 P1-1：预处理步骤完全硬编码**

用户无法选择或跳过任何步骤。5 个步骤强制依次执行，包括：
- 缺少 Dewow（去低频）：LPR 数据的天线耦合会产生显著的低频漂移，不做 dewow 会导致后续能量检测被低频污染
- 仅支持带通 FIR 滤波，不支持高通/低通
- 仅支持高斯 AGC，不支持标准 AGC 或 power gain
- 没有道间均衡化（equalize），导致某些异常强道主导能量直方图

**建议**：将预处理改为可配置模式，用户在 UI 中勾选需要的步骤。

---

### 步骤 2：振幅裁剪（行 1637-1641）

裁剪至 ±4。值合理但硬编码，应作为参数 `clip` 暴露（代码已支持但 UI 未暴露）。

---

### 步骤 3：能量图二维平滑（行 1643-1645）

```js
energy = |amplitude|
energySmoothed = smoothTraces(smoothSamples(energy, 17), 13)
```

**问题 P2-1：平滑参数硬编码**

- 时间方向 17 样点：对应 ~5.3ns（dt=0.3125ns），匹配 ~10ns 脉宽
- 空间方向 13 道：对应 ~0.65m（dx=0.05m）

这些值是为 CE-3 数据调优的，不适用于其他数据。应作为参数暴露。

---

### 步骤 4：逐道峰值检测（行 1652-1664）

```
每道：检测局部极大值 → 按能量排序 → 贪心选 top 4（间距 ≥ 0.8m）
```

**问题 P2-2：硬限制每道最多 4 个峰值**

如果实际有 5+ 个连续反射层，超出部分会被丢弃。应作为参数。

**问题 P2-3：贪心选择的偏向**

按能量排序后贪心选取，可能导致 4 个峰聚集在同一深度区间，而忽略较弱的真实层位。

---

### 步骤 5：直方图聚类（行 1667-1701）

```
0.5m bin → 72分位数阈值 → 检测直方峰 → 聚类(±0.75m) → 合并(<0.7m)
```

**问题 P2-4：多个阈值硬编码**

0.5m bin、72% 阈值、0.75m 搜索半径、0.7m 合并距离——全部不可配置。

---

### 步骤 6：种子层位筛选（行 1703-1705）

```
support > 22% × nt → 否则 fallback 取 top 6
```

**问题 P2-5：22% 支持度阈值**

仅适用于月面数据（层位连续性好）。陆地 GPR 数据中层位可能更短。应暴露为参数。

---

### ⚠️ 步骤 7：逐道追踪 — **核心缺陷**（行 1727-1742）

```js
for each seed horizon:
  search window = [center ± 0.85m]  // 逐道独立搜索
  line[t] = argmax(energy[t, s])     // 纯能量最大值
smoothLine(line, 61)                 // 后处理平滑
```

#### 🔴 问题 P0：追踪线与地质模型分层不贴合

**根本原因：逐道独立追踪 + 无层序约束 = 追踪线可交叉**

追踪线计算逻辑（行 1734-1740）：
```
对每道 t，在窗口 [s0, s1] 内独立搜索 energy 最大值
```

地质模型分层逻辑（行 1764-1770）：
```
对每点 (z, t)：layer = count(horizons where depth >= line[t])
```

**当两条追踪线交叉时：**
1. H1 在道 100 深度 5m，在道 200 深度 8m
2. H2 在道 100 深度 7m，在道 200 深度 6m
3. 两条线在道 100-200 之间交叉
4. 分层计数 `depth >= h.line[t]` 在交叉点产生不一致结果

**后果：** 模型栅格图中出现不连续的地质层边界，追踪线（剖面图上）与模型分层（栅格图）不匹配。

#### 🟡 问题 P1-2：无横向连续性约束

MATGPR 的 `markerinterp.m` 使用带平滑约束的层位插值，确保相邻道深度差不超过阈值。当前实现无此约束，单道能量最大值会导致追踪线跳动幅度达 ±0.85m。

#### 🟡 问题 P2-6：后处理平滑掩盖问题

61 点滑动平均（~3m 空间窗口）可以平滑掉跳变，但也会抹去真实的小尺度地形。

---

### 步骤 8：模型栅格化（行 1761-1771）

```
modelData[z*nt + t] = Σ(horizon.line[t] <= depth)
```

**当前实现依赖步骤 7 的追踪线**，继承了所有交叉和跳动问题。

## 追踪线与模型分层贴合的解决方案

### 方案：层序约束追踪 + 分层一致性验证

**核心思路**：追踪线强制保持深度顺序（H1_d < H2_d < H3_d < ...），确保分层计数永远一致。

实现要点：
1. 种子按深度排序后分配固定层序
2. 追踪时，对每道 t，同时追踪所有层位，加入约束：
   - `H_i_depth[t] < H_{i+1}_depth[t]`（强制最小间距 ≥ 1 样点）
3. 动态规划或贪心顺序追踪：
   - 先追踪 H1（最浅），H2 搜索窗口下界 = max(H2_center - window, H1_depth[t] + min_gap)
   - 依次追踪 H3, H4...
